How I Work

01 Introduction

Overall project overview

Every good project starts with good explanation and instruction. When you know exactly what you need i can make plan how to do this. Then i can give you proposal what is better for some solution.

Problem identification

When i have project instruction i always look for problem. Problem means that because you need my services (ex. make website), you need this to attract customers or sell product(something like that). If the problem is identified correctly i do my best to make project appropriet for this problem.

02 Prototyping

Create prototype

In this step the project starts to look like a real project. I make and send the first sketch to the customer (it can be sketch, wireframes or whatever we agree). This is necessary so if i missed something customer can make changes or just add some new in this.

Make agreement

After all the changes and confirmation that it is exactly as required, the last sketch is made. The final layout of the project is about 90% of the layout of the sketch itself. So this is the most important step.

03 Development

Production

When the coding begins there will generally be less communication from me as this is quite an involved process that doesn't require much feedback. This is just making according to sketches.

Testing

The whole project is ready, but we need test for be sure about everything work fine. In this step customer test and looks to see if any minor changes are needed. If necessary, I make the last changes and alterations so that everything is in ideal condition.

Launch

When it's over i'll take care of everything. I can just send project to customer, but what he can do if he don't know how to use it. That's why it's up to me to buy everything what you need, deploy project and make sure it is running.
